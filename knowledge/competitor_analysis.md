# Competitor Analysis

## Comparison Matrix

| 产品 | 定位 | 核心能力 | 缺失 |
|------|------|---------|------|
| README.so | README 生成器 | 模板/Badge/安装说明 | 不理解项目价值 |
| Mintlify | 代码→文档 | 自动文档/产品展示 | 不做审核模拟 |
| SonarQube | 代码质量 | 可维护性/可靠性/安全性 | 不改 README/文档 |
| Linear | 团队管理 | Issue→Project→Delivery | 不触及产品包装 |
| OpenHands | 自动开发 | 需求→读仓库→改代码→提交 | 无交付包装层 |
| Jules(Google) | 仓库理解 | 读取→理解→修改→PR | 不是评审系统 |
| Claude Code | 代码修改 | 仓库理解/代码修改/提交建议 | 无产品包装层 |

## BOS-FS Position
唯一把"生成→表达→审核→交付"串联成统一 Skill Runtime 的系统。
差异化关键词：Submission Engineering（提交工程）
