# README 评分标准

## 评分维度

| 维度 | 分值 | 说明 |
|------|------|------|
| 清晰度 | 0-10 | 表达是否简洁准确，读者能否快速理解项目 |
| 完整性 | 0-10 | 是否覆盖 What/Why/How/Result/Next 全部模块 |
| 专业性 | 0-10 | 术语使用、格式规范、架构图/表格是否专业 |
| 可操作性 | 0-10 | 快速开始、部署说明是否可直接执行 |

## 评分细则

### 清晰度 (0-10)

| 分数 | 标准 |
|------|------|
| 9-10 | 一句话精准概括，层次分明，无任何歧义 |
| 7-8 | 表达清楚，偶有冗余但不影响理解 |
| 5-6 | 基本能读懂，部分段落需要反复理解 |
| 3-4 | 表达混乱，需要猜测作者意图 |
| 0-2 | 无法理解项目是什么 |

### 完整性 (0-10)

| 分数 | 标准 |
|------|------|
| 9-10 | 5 个模块齐全且内容丰富 |
| 7-8 | 5 个模块齐全，个别模块内容较薄 |
| 5-6 | 缺失 1 个模块或部分模块严重缺失 |
| 3-4 | 仅覆盖 2-3 个模块 |
| 0-2 | 几乎无有效内容 |

### 专业性 (0-10)

| 分数 | 标准 |
|------|------|
| 9-10 | 架构图规范、表格清晰、术语准确、格式统一 |
| 7-8 | 整体专业，细节有小瑕疵 |
| 5-6 | 格式基本合格，缺乏图表支撑 |
| 3-4 | 格式混乱，术语使用不当 |
| 0-2 | 无基本排版，纯文字堆砌 |

### 可操作性 (0-10)

| 分数 | 标准 |
|------|------|
| 9-10 | 复制命令即可运行，环境要求明确，故障排查指引齐全 |
| 7-8 | 步骤清晰，个别细节需自行推断 |
| 5-6 | 有基本步骤，但缺少环境说明或依赖信息 |
| 3-4 | 步骤不完整，无法独立运行 |
| 0-2 | 无可执行内容 |

## 总分解读

| 总分区间 | 等级 | 解读 |
|---------|------|------|
| 36-40 | 优秀 | 可直接用于正式提交 |
| 28-35 | 良好 | 小幅修改后可用 |
| 20-27 | 及格 | 需要较大改进 |
| 0-19 | 不合格 | 需要重写 |
