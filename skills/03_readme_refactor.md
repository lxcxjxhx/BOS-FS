# README Refactor Skill

## Purpose
将原始 README 重构为 What/Why/How/Result/Next 结构。

## Activation
当需要优化项目文档、提高可理解度时激活。

## Input
原始 README 文本或仓库信息。

## Output Structure
```
# [项目名称]

## What
一句话价值：[技术 × 用户 × 收益]

## Why
- 痛点：[用户面临的问题]
- 现有方案的不足：[对比分析]

## How
- 核心架构：[ASCII 架构图]
- 关键特性：[3-5 个]
- 快速开始：[安装/使用步骤]

## Result
- 指标：[可量化的收益]
- 场景：[典型使用场景]

## Next
- 路线图：[近期/中期/远期规划]
```

## Value Formula
```
技术 × 用户 × 收益 = 产品描述
```

## Transformation Rules
- "AI Workflow Engine" → "帮助开发者将需求自动转换为可交付软件资产的平台"
- "支持多模型" → "减少重复配置与上下文切换"
- "模型调度" → "降低交付成本"
- 技术术语 → 用户价值
- 功能列表 → 场景描述

## Example
Input: "# My Project\n\nAn AI workflow engine that supports multi-model scheduling."
Output: 按上述结构重构后的完整 README
