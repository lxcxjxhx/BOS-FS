# Reject Analyzer Skill

## Purpose
分析项目被拒绝的原因，提取真实问题，给出修复建议。

## Activation
当项目被拒、需要分析原因并重投时激活。

## Input
拒绝原因文本。

## Common Reject Patterns
| 拒绝原因 | 真实问题 | 修复方向 |
|---------|---------|---------|
| "表达太底层" | 用户价值弱 | 从工具 → 交付平台 |
| "目标不明确" | 场景缺失 | 增加典型使用场景 |
| "又一个 Agent 套壳" | 差异化不足 | 突出独特价值主张 |
| "文档不完整" | 交付物不全 | 补齐 Demo/FAQ/风险说明 |
| "安全风险" | 缺少安全设计 | 补充安全评估报告 |

## Output Format
```json
{
  "real_issue": "真实问题描述",
  "fixable_items": ["可修改项1", "可修改项2"],
  "resubmit_suggestion": "重新提交版本建议"
}
```

## Rules
- 分析拒绝原因背后的真实问题
- 给出具体可操作的修复项
- 始终附加通用检查项（README/场景/指标/风险）
